# 小Y项目需求分析说明书

## 1. 文档说明

本目录用于沉淀“小Y”校园智能生活服务平台的需求分析成果。  
其中：

- `xiaoy-ui` 为前端项目，技术栈为 `React + TypeScript + Vite`
- `xiaoY` 为后端项目，技术栈为 `Spring Boot 3 + MyBatis-Plus + MySQL + MongoDB + Redis + MinIO + LangChain4j`

本文档结合当前代码目录、页面结构、接口命名和已实现模块进行整理，可作为：

1. 项目立项说明材料
2. 课程设计/毕业设计需求文档
3. 前后端联调说明基础
4. 后续详细设计、接口设计、测试设计的输入文档

## 2. 项目定位

小Y是一个面向校园场景的智能服务平台，覆盖“学习 + 生活 + 问答 + 管理”四类能力，核心目标包括：

- 为学生提供统一的信息入口
- 提供课程、食堂、菜品、评价等校园高频信息服务
- 提供AI聊天能力，提高信息获取效率
- 提供后台管理能力，支持内容维护与数据治理

## 4. 项目当前识别到的核心功能

根据现有代码结构，系统已覆盖或计划覆盖如下能力：

### 用户侧能力

- 用户注册、登录、退出
- 登录态持久化
- 个人信息查看与编辑
- 头像上传
- 首页内容展示
- 课程信息浏览
- 课程评价查看与提交
- 食堂位置与菜品查看
- 校车与校园生活信息浏览
- AI聊天与会话记录

### 管理侧能力

- 用户管理
- 课程管理
- 课程评论管理
- 食堂地点管理
- 菜品管理

### 后端支撑能力

- REST接口服务
- 数据库读写与分页查询
- Token鉴权拦截
- Redis登录态缓存
- MongoDB会话记忆存储
- MinIO文件上传
- AI问答与流式输出

5. 最后看 `14`，用于汇报展示、答辩讲解和图形化说明
