# 业务背景与目标

## 1. 业务背景

校园场景中的信息通常存在以下问题：

1. **信息分散**：课程、食堂、菜品、评价、个人资料等信息散落在不同系统或非结构化渠道中
2. **查询成本高**：学生需要在多个页面或群消息中反复查找
3. **信息更新不统一**：缺乏统一维护机制，内容容易过期
4. **交互方式单一**：传统系统以列表查询为主，缺少智能问答能力
5. **运维管理薄弱**：缺少统一后台对课程、评论、地点、菜品等内容进行维护

基于以上背景，需要建设一个面向校园服务的一体化平台，将学习与生活信息集中管理，并通过AI能力提升使用体验。

## 2. 建设目标

### 2.1 总体目标

建设一个“可查询、可管理、可交互、可扩展”的校园智能服务平台。

### 2.2 分项目标

#### 用户服务目标

- 支持用户注册、登录和个人信息管理
- 支持课程、食堂、菜品、评价等信息查询
- 支持校园生活信息浏览
- 支持通过自然语言向AI提问

#### 管理目标

- 支持管理员维护核心业务数据
- 支持对课程、菜品、地点、评论、用户进行统一管理
- 降低数据维护的人工成本

#### 技术目标

- 建立前后端分离系统
- 提供清晰接口边界和可维护数据结构
- 支持流式AI对话、文件上传和基础鉴权
- 便于后续继续拓展推荐、知识库、移动端等能力

## 3. 预期收益

### 3.1 对普通用户

- 查询信息更集中
- 使用路径更简洁
- 可通过AI快速获得答案
- 获得更好的校园服务体验

### 3.2 对管理员

- 能够集中管理业务数据
- 提升内容维护效率
- 降低数据错漏和信息过期问题

### 3.3 对项目建设

- 形成完整的校园服务平台原型
- 为后续产品化、展示、答辩或扩展开发打下基础

## 4. 项目成功标志

若满足以下条件，可认为项目达成阶段目标：

1. 前后端可独立启动并完成联调
2. 核心页面可访问，主要功能可正常操作
3. 用户可完成登录、资料查看、课程查询、评论查看、AI问答等操作
4. 管理员可完成基础数据维护
5. 系统具备基础部署和演示能力
