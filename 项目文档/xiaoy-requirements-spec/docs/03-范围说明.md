# 范围说明

## 1. 本期建设范围

结合当前仓库代码与已存在页面结构，本项目本期建设范围如下。

### 1.1 前端范围

前端 `xiaoy-ui` 负责以下内容：

- 首页与导航展示
- 用户登录与注册页面
- 用户资料页
- 课程页
- 课程评价页及评价详情页
- 食堂页
- 校车页
- 校园生活页
- AI聊天页
- 管理后台页面
- 文件上传相关页面交互

### 1.2 后端范围

后端 `xiaoY` 负责以下内容：

- 用户注册、登录、用户信息查询和修改
- 课程数据查询与维护
- 课程评论查询、提交与维护
- 食堂地点查询与维护
- 菜品查询与维护
- 文件上传
- 聊天会话列表、消息查询、流式聊天
- 登录鉴权与Token校验
- AI模型调用与会话记忆存储

### 1.3 数据范围

本项目当前覆盖或涉及的主要数据实体包括：

- 用户 `User`
- 用户信息 `UserInfo`
- 课程 `Course`
- 课程评论 `CourseComment`
- 菜品 `Dish`
- 食堂地点 `CanteenLocation`
- 聊天消息/会话 `ChatMessages`

## 2. 本期不纳入范围

为保证项目聚焦，本期不强制纳入以下内容：

1. 第三方支付能力
2. 完整移动端App开发
3. 复杂推荐系统
4. 多租户权限体系
5. 全量运营数据分析平台
6. 完整消息通知中心
7. 高并发分布式拆分改造

## 3. 范围边界说明

### 3.1 已有能力边界

当前系统更偏向于“校园服务平台原型/业务基础版”，重点是：

- 先打通主要业务流程
- 先建立完整页面与接口骨架
- 先形成可演示、可运行、可扩展的系统基础

### 3.2 信息类页面边界

如“校车”“生活服务”等页面，本期更偏向信息展示能力，是否完全由后端动态驱动，可在后续版本逐步完善。

### 3.3 AI能力边界

本项目已具备AI聊天基础，但本期不要求达到通用问答平台级别，而是面向校园场景实现：

- 基础问答
- 流式输出
- 会话记忆
- 可扩展知识增强

## 4. 交付物范围

本项目交付物包括：

1. 前端项目源码
2. 后端项目源码
3. 需求分析文档
4. 接口与数据说明文档
5. 部署运行说明
6. 可演示的运行版本

## 5. 约束条件

### 5.1 技术约束

- 前端基于 React 技术栈开发
- 后端基于 Java Spring Boot 开发
- 数据库使用 MySQL，部分会话数据使用 MongoDB
- 登录态依赖 Redis
- 文件存储依赖 MinIO
- AI能力依赖外部模型服务配置

### 5.2 环境约束

- 当前主要开发环境为 Windows
- 本地开发默认通过 `localhost` 进行前后端联调
- 后端运行依赖数据库、中间件及模型配置可用
