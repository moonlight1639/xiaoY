# 部署与运行要求

## 1. 开发环境要求

### 1.1 前端环境

- Node.js 18 及以上版本为宜
- npm 或 pnpm 可用于依赖管理
- 现代浏览器用于本地调试

### 1.2 后端环境

- JDK 17
- Maven 3.8 及以上版本
- Spring Boot 3 运行环境

### 1.3 中间件环境

后端运行依赖以下组件：

- MySQL：存储业务数据
- Redis：存储 token 和登录态
- MongoDB：存储聊天记忆与会话数据
- MinIO：存储上传文件
- 外部模型服务：用于AI聊天和嵌入能力

## 2. 本地运行要求

## 2.1 前端运行要求

- 默认通过 Vite 启动开发服务
- 前端请求地址当前指向 `http://localhost:8080/xiaoY`
- 若后端地址变更，需同步调整前端请求配置

## 2.2 后端运行要求

- 启动前需确保 MySQL、Redis、MongoDB、MinIO 可访问
- `application.yaml` 中的主机、账号、密码、模型Key需正确配置
- 若 AI 服务未配置完成，聊天相关接口可能不可用

## 3. 配置要求

## 3.1 数据库配置

需在配置文件中明确：

- MySQL 地址、端口、数据库名
- 用户名、密码
- 时区、字符集等连接参数

## 3.2 Redis 配置

需配置：

- Redis 主机
- Redis 端口

## 3.3 MongoDB 配置

需配置：

- MongoDB 连接字符串
- 聊天数据库名称

## 3.4 MinIO 配置

需配置：

- MinIO 服务地址
- Access Key
- Secret Key
- Bucket 名称
- 文件前缀与访问有效期

## 3.5 AI 模型配置

当前配置文件中涉及：

- DeepSeek 兼容接口配置
- 通义千问聊天模型配置
- 通义千问流式模型配置
- 通义千问嵌入模型配置

运行前需保证相关 API Key 已正确注入环境变量。

## 4. 部署要求

## 4.1 前端部署

- 构建产物应可部署到静态 Web 服务器
- 建议通过 Nginx 提供静态资源访问
- 需正确配置前端路由重写

## 4.2 后端部署

- 后端可打包为 jar 后独立运行
- 建议通过 Linux 服务器部署并结合进程守护管理
- 应开放前端访问所需端口和跨域策略

## 5. 联调要求

- 前端与后端接口前缀需保持一致
- 跨域配置需允许前端访问
- 认证头字段需统一使用 `Authorization`
- 流式聊天接口需支持前端 `text/event-stream` 或流式读取方式

## 6. 日志与监控要求

- 后端需输出基础调试日志，便于排查问题
- 部署环境应保留运行日志
- 关键异常包括数据库连接失败、Redis失效、模型调用失败、上传失败等应能定位

## 7. 运行验证要求

部署完成后，应至少验证以下内容：

1. 前端首页可访问
2. 用户可完成注册和登录
3. 课程、评论、食堂、菜品接口可正常返回数据
4. AI聊天接口可返回消息
5. 上传接口可正常使用
6. 管理页面可完成基本数据维护
