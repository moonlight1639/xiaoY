# xiaoY 后端需求

## 1. 模块定位

`xiaoY` 是小Y项目的业务与服务支撑层，负责：

- 提供 REST 接口
- 处理核心业务逻辑
- 访问和维护数据库数据
- 实现登录鉴权
- 支持文件上传
- 提供AI问答与流式会话能力

后端当前基于 `Spring Boot 3 + MyBatis-Plus + MySQL + Redis + MongoDB + MinIO + LangChain4j`。

## 2. 接口基础要求

### 2.1 接口风格

- 接口统一使用 HTTP/JSON 进行交互
- 路径统一挂载在 `/xiaoY` 前缀下
- 查询类接口以 `GET` 为主
- 新增类接口以 `POST` 为主
- 更新类接口以 `PUT` 为主
- 删除类接口以 `DELETE` 为主

### 2.2 响应要求

- 响应应统一封装成功状态、错误消息、数据体和总数等字段
- 出现业务错误时应返回可理解的错误信息

## 3. 业务功能需求

## 3.1 用户账号模块

### 3.1.1 用户注册

- 提供用户注册接口
- 校验用户名是否可用
- 保存用户基础登录信息
- 与用户资料建立关联关系

### 3.1.2 用户登录

- 校验用户名和密码
- 登录成功后生成或返回可用于鉴权的 `token`
- 将登录态与用户信息缓存到 Redis

### 3.1.3 用户信息查询

- 支持用户列表分页查询
- 支持按ID获取用户信息

### 3.1.4 用户信息维护

- 支持新增、修改、删除用户基础信息

## 3.2 用户资料模块

### 3.2.1 用户资料查询

- 支持分页查询用户资料列表
- 支持按ID获取用户资料详情

### 3.2.2 用户资料修改

- 支持修改昵称、手机号、邮箱、头像、性别、状态、用户类型等字段

### 3.2.3 头像上传

- 提供头像上传接口
- 校验文件不能为空
- 上传成功后更新对应资源访问地址或记录

## 3.3 课程模块

### 3.3.1 课程查询

- 支持课程列表分页查询
- 支持按ID查询课程详情

### 3.3.2 课程维护

- 支持新增课程
- 支持修改课程信息
- 支持删除课程

### 3.3.3 课程字段要求

课程至少应支持以下信息：

- 课程名称
- 任课教师
- 简介/正文内容
- 封面图
- 阅读量、点赞量、评论量、收藏量等统计字段

## 3.4 课程评论模块

### 3.4.1 评论查询

- 支持评论列表分页查询
- 支持按课程ID查询评论集合

### 3.4.2 评论提交

- 登录用户可提交课程评论
- 系统应记录课程ID、用户ID、用户名、头像和评论内容等信息

### 3.4.3 评论维护

- 支持更新评论
- 支持删除评论
- 管理员可进行评论治理

## 3.5 食堂地点模块

### 3.5.1 地点查询

- 支持食堂地点列表分页查询
- 支持按ID查询地点信息

### 3.5.2 地点维护

- 支持新增地点
- 支持修改地点
- 支持删除地点

### 3.5.3 关联展示

- 支持返回地点与菜品展示相关的辅助数据

## 3.6 菜品模块

### 3.6.1 菜品查询

- 支持菜品列表分页查询
- 支持按ID查询菜品详情

### 3.6.2 菜品维护

- 支持新增菜品
- 支持修改菜品
- 支持删除菜品

### 3.6.3 菜品字段要求

菜品至少应包含：

- 菜品名称
- 描述
- 图片
- 价格
- 分类
- 所属地点ID和名称
- 状态与点赞数等字段

## 3.7 文件上传模块

### 3.7.1 通用文件上传

- 提供文件上传接口
- 接收文件和目标路径参数
- 校验文件非空
- 上传到对象存储并返回访问地址

### 3.7.2 上传支撑要求

- 支持 MinIO 存储
- 支持后续扩展到更多文件类型

## 3.8 AI聊天模块

### 3.8.1 普通聊天接口

- 提供标准问答接口
- 接收 `memoryId` 与用户输入内容
- 返回AI生成结果

### 3.8.2 会话列表接口

- 支持返回会话标题列表
- 支持按会话ID查询消息历史

### 3.8.3 新会话创建

- 支持根据首条消息生成新会话ID
- 支持生成默认标题与创建时间

### 3.8.4 流式聊天

- 提供流式返回接口
- 逐段输出AI回答内容
- 前端可按分片实时渲染结果

### 3.8.5 会话记忆

- 聊天历史应支持持久化
- 会话数据可保存在 MongoDB
- 后续支持继续接入知识增强能力

## 3.9 智能填充模块

- 提供根据上下文、目标类信息生成结构化内容的能力
- 用于辅助生成类内容或结构化信息填充
- 作为AI增强能力保留扩展入口

## 4. 鉴权与安全需求

## 4.1 Token鉴权

- 受保护接口应校验 `Authorization` 请求头
- 若 token 不存在或过期，应返回未登录或重新登录提示

## 4.2 拦截范围

当前至少应对以下接口进行鉴权控制：

- 聊天相关接口
- 评论提交接口

## 4.3 安全处理

- 所有输入参数应进行必要校验
- 文件上传需控制空文件与非法路径风险
- 敏感信息不应直接暴露在响应中

## 5. 数据与存储需求

### 5.1 MySQL

用于存储结构化业务数据：

- 用户
- 用户资料
- 课程
- 课程评论
- 菜品
- 食堂地点

### 5.2 Redis

用于存储：

- 登录态映射
- token 与用户信息缓存

### 5.3 MongoDB

用于存储：

- 聊天会话
- 消息历史
- AI记忆相关数据

### 5.4 MinIO

用于存储：

- 头像资源
- 业务附件
- 其他上传文件

## 6. 可维护性要求

- Controller、Service、Mapper 分层清晰
- 实体字段命名与前端类型尽量保持一致
- Mapper XML 与数据库结构保持同步
- 后续应补充接口文档和字段说明
