# xiaoy-ui 前端需求

## 1. 模块定位

`xiaoy-ui` 是小Y项目的用户交互层，负责：

- 页面展示
- 路由管理
- 用户交互处理
- 前后端接口联调
- 登录态维护
- 聊天流式响应展示
- 管理后台页面承载

前端当前基于 `React + TypeScript + Vite`，并使用 Zustand 管理认证状态。

## 2. 页面与路由需求

根据当前路由实现，前端应至少支持以下页面：

| 路由 | 页面 | 说明 |
|---|---|---|
| `/` | 首页 | 系统门户页 |
| `/chat` | AI聊天页 | 聊天会话与流式回复 |
| `/courses` | 课程页 | 课程列表展示 |
| `/life` | 生活服务页 | 校园生活信息展示 |
| `/reviews` | 评价页 | 课程评价列表 |
| `/reviews/info/:id?` | 评价详情页 | 指定评价信息展示 |
| `/canteen` | 食堂页 | 食堂地点与菜品信息 |
| `/bus` | 校车页 | 校车信息展示 |
| `/userinfo` | 个人信息页 | 用户资料维护 |
| `/aitest` | AI测试页 | AI能力调试或试验页面 |
| `/admin/*` | 管理后台 | 用户、课程、评论、地点、菜品管理 |
| `*` | 404页 | 路由兜底 |

## 3. 核心功能需求

## 3.1 认证模块

### 3.1.1 注册

- 提供注册表单
- 至少包含用户名、密码等必填项
- 提交后调用后端注册接口
- 根据结果显示成功或失败提示

### 3.1.2 登录

- 提供登录表单
- 登录成功后保存 `token` 和用户信息
- 登录态应支持本地持久化
- 后续接口请求应自动附带认证信息

### 3.1.3 退出登录

- 支持清除本地登录状态
- 清除缓存中的 `token`
- 回到未登录可访问状态

## 3.2 首页模块

- 展示平台名称、主要模块入口和导航
- 提供课程、食堂、AI聊天等高频功能快捷入口
- 页面风格应统一、易于理解

## 3.3 用户信息模块

### 3.3.1 用户信息展示

- 显示昵称、手机号、邮箱、性别、头像、用户类型、状态等信息
- 支持从后端获取用户资料并回显

### 3.3.2 用户信息编辑

- 支持编辑基础资料
- 表单提交前应做基础校验
- 保存后应刷新页面数据或给出成功提示

### 3.3.3 头像上传

- 支持上传头像文件
- 上传完成后显示最新头像
- 上传失败需给出错误提示

## 3.4 课程模块

### 3.4.1 课程列表展示

- 展示课程名称、教师、描述、阅读量、点赞量、评论量等字段
- 支持从后端加载课程数据

### 3.4.2 课程信息浏览

- 用户可查看课程简介和详情内容
- 页面需保证列表展示清晰

## 3.5 评论模块

### 3.5.1 评论列表

- 展示课程评论内容、评论用户、头像、点赞/点踩数、时间等信息
- 支持按课程查看评论

### 3.5.2 评论提交

- 已登录用户可提交课程评论
- 提交前需校验评论内容非空
- 提交成功后刷新评论列表

### 3.5.3 评论详情

- 进入评价详情页时应根据路由参数加载对应内容

## 3.6 食堂与菜品模块

### 3.6.1 食堂地点展示

- 展示食堂名称或地点信息
- 提供地点列表或分组导航

### 3.6.2 菜品展示

- 展示菜品名称、价格、分类、描述、图片、所属地点等信息
- 支持按地点查看菜品信息

## 3.7 校车与生活服务模块

- 提供校园生活类信息页面
- 提供校车类信息页面
- 当前可优先实现静态/半动态展示

## 3.8 AI聊天模块

### 3.8.1 会话列表

- 支持加载历史会话标题列表
- 支持点击会话标题查看消息历史
- 支持新建会话
- 支持删除前端本地会话视图

### 3.8.2 聊天消息展示

- 区分用户消息和AI消息
- 支持 Markdown 渲染
- 支持自动滚动到底部
- 流式输出期间应尽量减少闪烁

### 3.8.3 流式请求交互

- 发送消息时，前端应调用流式聊天接口
- 接收分片内容并实时拼接显示
- 会话结束后将完整AI回复落入当前消息列表

### 3.8.4 登录校验

- 未登录用户进入聊天发送时，应提示先登录
- 受保护接口调用失败时，应给出清晰错误反馈

## 3.9 后台管理模块

### 3.9.1 管理后台布局

- 提供左侧菜单导航与主内容区
- 支持模块切换与当前菜单高亮

### 3.9.2 用户管理

- 展示用户列表
- 支持新增、编辑、删除或维护用户信息

### 3.9.3 课程管理

- 展示课程列表
- 支持课程增删改查

### 3.9.4 评论管理

- 展示评论列表
- 支持评论维护与删除

### 3.9.5 地点管理

- 展示食堂地点列表
- 支持地点增删改查

### 3.9.6 菜品管理

- 展示菜品列表
- 支持菜品增删改查

## 4. 公共交互需求

## 4.1 请求处理

- 应封装统一的 HTTP 请求实例
- 自动附带 `Authorization` 请求头
- 统一处理接口错误和超时情况

## 4.2 状态管理

- 用户认证信息应统一管理
- 页面刷新后应尽量保留登录状态

## 4.3 表单体验

- 输入项应具备必填和格式校验
- 提交过程应有加载和结果反馈

## 4.4 页面体验

- 页面结构清晰
- 信息层级明确
- 管理页面操作结果可感知

## 5. 前端边界说明

前端主要负责页面表现与交互流程，不承载核心业务规则判定，核心权限、数据校验、文件处理和会话存储应由后端负责。
