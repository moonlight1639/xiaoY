# 接口与数据需求

## 1. 接口设计原则

### 1.1 基本原则

- 前后端通过 HTTP + JSON 进行数据交互
- 后端接口统一以 `/xiaoY` 为服务前缀
- 返回结构建议统一为：`success`、`errorMsg`、`data`、`total`
- 分页列表接口统一接收 `pageNum`、`pageSize`

### 1.2 鉴权原则

- 受保护接口在请求头中传递 `Authorization`
- 当前聊天相关接口和评论提交接口需要登录后访问

## 4. 存储需求

### 4.1 MySQL表范围

- 用户表
- 用户资料表
- 课程表
- 课程评论表
- 食堂地点表

- 聊天会话集合
- 聊天消息集合

### 4.3 Redis键值范围

- `token -> 用户信息` 映射

## 5. 数据一致性要求

- 前后端字段命名应尽量统一
- 课程与评论、地点与菜品之间应保持关联一致性
- 删除操作应考虑逻辑删除字段或关联影响
- 会话ID在单个聊天上下文中必须唯一
