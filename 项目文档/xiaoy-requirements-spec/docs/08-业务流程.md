# 业务流程

## 1. 用户注册与登录流程

1. 用户进入首页或登录组件
2. 若无账号，先进入注册组件填写用户名、密码等信息
3. 前端调用注册接口提交数据
4. 后端校验并保存用户数据
5. 用户返回登录页输入账号密码
6. 前端调用登录接口
7. 后端校验成功后返回用户资料和 `token`
8. 前端保存登录态并进入系统主流程

## 2. 用户资料查看与维护流程

1. 用户进入个人信息页
2. 前端读取当前登录用户信息并请求资料详情
3. 页面展示昵称、邮箱、手机号、头像等数据
4. 用户编辑资料并提交
5. 前端调用更新接口
6. 后端保存修改结果并返回成功状态
7. 前端刷新页面数据

## 3. 课程浏览流程

1. 用户进入课程页
2. 前端请求课程列表接口
3. 后端返回课程数据
4. 前端展示课程名称、教师、简介等内容
5. 用户可进一步查看课程相关评论信息

## 4. 课程评论流程

### 4.1 评论查看

1. 用户进入评价页
2. 前端请求课程评论列表或按课程ID查询评论
3. 后端返回评论数据
4. 前端展示评论内容、用户信息和时间

### 4.2 评论提交

1. 已登录用户输入评论内容
2. 前端携带 `Authorization` 调用评论提交接口
3. 后端从登录态中识别用户身份
4. 后端保存评论并返回成功结果
5. 前端刷新评论列表

## 5. 食堂与菜品浏览流程

1. 用户进入食堂页
2. 前端请求地点列表及地点-菜品辅助数据
3. 后端返回食堂地点和菜品信息
4. 前端按地点或分类进行展示
5. 用户浏览菜品名称、价格、图片和描述

## 6. AI聊天流程

### 6.1 历史会话加载

1. 用户进入聊天页
2. 前端请求会话标题列表
3. 后端返回历史会话信息
4. 前端渲染左侧会话列表

### 6.2 新建会话并发送消息

1. 用户点击新对话或直接输入问题
2. 若当前无会话ID，前端先请求创建新会话
3. 后端生成 `memoryId`、标题和创建时间
4. 前端保存新会话并展示在会话列表中
5. 用户发送消息
6. 前端将用户消息先写入当前会话视图

### 6.3 流式回答

1. 前端调用流式聊天接口
2. 后端基于AI模型逐段返回回答内容
3. 前端持续拼接响应片段并实时渲染
4. 流式结束后，前端将完整AI回复追加进消息列表
5. 会话数据在后端持久化存储

### 6.4 查看历史消息

1. 用户点击某条历史会话标题
2. 前端按 `memoryId` 请求消息历史
3. 后端返回该会话下的消息集合
4. 前端加载并显示完整对话记录

## 7. 管理后台流程

### 7.1 管理员进入后台

1. 管理员进入 `/admin`
2. 前端显示后台侧边栏导航
3. 管理员切换不同管理模块

### 7.2 用户管理流程

1. 请求用户或用户资料列表
2. 展示分页数据
3. 执行新增、编辑、删除操作
4. 刷新列表并提示结果

### 7.3 课程管理流程

1. 请求课程列表
2. 新增或修改课程信息
3. 保存后刷新数据

### 7.4 评论管理流程

1. 请求评论列表
2. 删除不合规评论
3. 返回操作结果

### 7.5 地点与菜品管理流程

1. 请求地点或菜品列表
2. 执行增删改查
3. 保存后更新前台可见数据

## 8. 文件上传流程

1. 用户或管理员选择文件
2. 前端以表单方式提交文件
3. 后端校验文件不为空
4. 后端上传到 MinIO 或目标存储路径
5. 返回文件访问地址或上传成功结果

## 9. 异常处理流程

- 当用户未登录访问受保护接口时，后端返回“请先登录”
- 当 token 失效时，后端返回“token无效或已过期”
- 当前端请求失败时，应统一提示服务器错误或接口错误信息
