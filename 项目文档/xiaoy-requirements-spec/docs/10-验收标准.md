# 验收标准

## 1. 功能验收标准

### 1.1 用户端验收

系统应至少满足以下功能：

1. 用户可完成注册、登录和退出
2. 用户可查看并修改个人资料
3. 用户可查看课程列表
4. 用户可查看课程评论并提交评论
5. 用户可查看食堂地点和菜品信息
6. 用户可进入AI聊天页面并完成问答
7. 用户可查看校园生活和校车等信息页面

### 1.2 管理端验收

管理员应至少可以：

1. 进入后台管理页面
2. 查看并维护用户数据
3. 查看并维护课程数据
4. 查看并维护课程评论
5. 查看并维护食堂地点
6. 查看并维护菜品信息

### 1.3 后端能力验收

后端应至少满足：

1. 核心接口可正常访问
2. 接口返回结构统一
3. 鉴权接口可识别登录状态
4. 聊天接口可返回正常结果
5. 文件上传接口可返回成功结果或有效地址

## 2. 联调验收标准

1. 前后端接口路径一致
2. 前端能正确展示后端返回数据
3. 受保护接口在登录后可调用，未登录时有明确提示
4. 流式聊天可在前端实时展示

## 3. 数据验收标准

1. 用户、课程、评论、地点、菜品等主要数据可正确查询
2. 新增、编辑、删除操作可正确反映到页面
3. 聊天历史会话可正确读取
4. 上传文件后可得到有效结果反馈

## 4. 性能验收标准

1. 常规页面在本地和测试环境下应具备可接受响应速度
2. 常规查询接口应稳定返回，不出现大面积超时
3. AI流式回答应能在较短时间内返回首段内容

## 5. 安全验收标准

1. 未登录用户无法访问受保护接口
2. 无效 token 调用接口时应返回错误提示
3. 上传空文件时应被后端拒绝
4. 用户敏感信息不应在前端明文暴露

## 6. 部署验收标准

1. 前端项目可正常启动和构建
2. 后端项目可正常启动和打包
3. MySQL、Redis、MongoDB、MinIO 配置正确时系统可稳定运行
4. 核心功能在目标环境中可复现演示

## 7. 文档验收标准

1. 需求文档结构完整
2. 接口与数据说明清晰
3. 部署运行说明可指导开发者完成本地启动
4. 文档内容与当前项目实现基本一致

## 8. 缺陷验收标准

满足以下条件可进入验收通过状态：

- 无阻塞性严重错误
- 核心流程可完整演示
- 主要功能无明显逻辑缺失
- 页面与接口能够支撑项目展示、答辩或继续开发
