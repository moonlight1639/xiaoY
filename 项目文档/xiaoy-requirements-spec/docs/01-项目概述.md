# 项目概述

## 1. 项目名称

小Y校园智能生活服务平台

## 2. 项目简介

小Y项目是一个围绕校园“学习信息、生活服务、智能问答、后台管理”建设的前后端分离系统。  
项目由两个核心子系统组成：

- `xiaoy-ui`：Web前端，负责页面展示、交互逻辑、状态管理和接口调用
- `xiaoY`：Java后端，负责业务处理、数据访问、鉴权、上传和AI能力支持

系统面向学生用户与管理员用户，目标是在单一入口内整合课程、食堂、菜品、评价、用户信息及AI问答服务。

## 3. 建设目标

### 3.1 业务目标

1. 为学生提供统一的校园信息服务入口
2. 提高课程、食堂、评论等信息的可查询性和可访问性
3. 借助AI对话提升信息获取效率
4. 为管理员提供基础数据维护后台

### 3.2 技术目标

1. 形成可运行的前后端分离架构
2. 建立可扩展的数据模型和接口体系
3. 支持聊天会话、文件上传与基础鉴权
4. 支持后续继续扩展更多校园应用场景

## 4. 总体架构概览

### 4.1 前端架构

前端基于 `React + TypeScript + Vite` 实现，包含：

- 页面路由组织
- 公共布局组件
- Zustand状态管理
- Axios接口请求封装
- 聊天页面流式输出展示
- 管理端页面模块化组织

### 4.2 后端架构

后端基于 `Spring Boot 3` 实现，包含：

- Controller：对外提供REST接口
- Service：封装业务逻辑
- Mapper/XML：访问MySQL数据
- Redis：登录态与缓存支撑
- MongoDB：聊天记忆与历史存储
- MinIO：文件上传与对象存储
- LangChain4j：AI问答与流式对话能力

## 5. 主要业务模块

### 5.1 用户与身份模块

- 用户注册
- 用户登录
- Token登录态维护
- 用户资料展示与修改
- 头像上传

### 5.2 校园内容模块

- 课程信息展示
- 课程评价展示与提交
- 食堂地点展示
- 菜品信息展示
- 校车与生活服务页面展示

### 5.3 智能问答模块

- 发起AI聊天
- 维护会话列表
- 查看历史消息
- 流式接收AI回复

### 5.4 后台管理模块

- 用户管理
- 课程管理
- 评论管理
- 地点管理
- 菜品管理

## 6. 项目价值

本项目的主要价值体现在以下方面：

1. **统一入口**：减少校园信息分散带来的查询成本
2. **交互友好**：通过Web前端集中承载各类服务
3. **管理可控**：通过后台管理维护核心业务数据
4. **智能增强**：结合AI问答提高信息检索效率
5. **可扩展**：具备后续接入更多校园模块的基础能力
