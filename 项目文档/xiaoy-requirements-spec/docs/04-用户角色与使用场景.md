# 用户角色与使用场景

## 1. 用户角色划分

## 1.1 普通用户

普通用户主要面向校园学生或一般使用者，负责使用平台提供的各类校园服务。

### 主要诉求

- 快速登录系统
- 查看个人资料
- 查询课程和课程评价
- 浏览食堂位置与菜品信息
- 查看校园生活/校车等信息
- 使用AI聊天进行咨询

### 典型权限

- 可访问用户端页面
- 可提交评论
- 可发起AI聊天
- 可维护自己的资料

## 1.2 管理员

管理员面向平台维护人员，负责业务数据的增删改查与内容治理。

### 主要诉求

- 登录后台管理页面
- 查看用户列表
- 维护课程、评论、地点、菜品等数据
- 保障平台内容可用性和准确性

### 典型权限

- 访问后台管理模块
- 查询与修改核心业务数据
- 进行批量或单条维护操作

## 1.3 系统运维/开发人员

该角色主要面向项目维护者，关注系统运行、配置和部署。

### 主要诉求

- 配置数据库、中间件和模型服务
- 启动与部署前后端系统
- 监控日志与排查问题
- 维护接口与服务稳定性

## 2. 典型使用场景

## 2.1 用户注册与登录场景

1. 用户打开首页
2. 点击登录或注册入口
3. 输入用户名和密码完成注册或登录
4. 系统返回用户信息和登录态
5. 用户进入系统继续使用其他模块

## 2.2 查看课程与评论场景

1. 用户进入课程模块
2. 查看课程列表
3. 选择某一课程查看详情或评价
4. 进入评价模块浏览已有评论
5. 登录用户可提交新的课程评价

## 2.3 查看食堂与菜品场景

1. 用户进入食堂模块
2. 查看食堂地点列表
3. 按地点查看菜品信息
4. 浏览菜品图片、价格、描述等内容

## 2.4 AI问答场景

1. 用户进入聊天页面
2. 系统加载历史会话列表
3. 用户选择已有会话或新建会话
4. 用户输入问题并发送
5. 后端流式返回AI回答
6. 系统保存会话标题和消息记录

## 2.5 用户资料维护场景

1. 用户进入个人信息页面
2. 查看昵称、手机号、邮箱、头像等信息
3. 修改字段后提交
4. 系统保存更新结果

## 2.6 管理员维护课程场景

1. 管理员进入后台
2. 进入课程管理模块
3. 查看课程列表
4. 对课程进行新增、修改或删除
5. 系统返回操作结果

## 2.7 管理员维护评论场景

1. 管理员进入评论管理模块
2. 查看课程评论列表
3. 删除不合规评论或修正相关内容

## 2.8 管理员维护地点与菜品场景

1. 管理员进入地点管理或菜品管理模块
2. 查看现有数据
3. 执行新增、修改、删除操作
4. 系统同步更新前台展示数据
